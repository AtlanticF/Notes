关于如何 review code
---
- 请求合并的分支是否正确
- mr 中是否有相关 issue 的描述以及改动点描述
- commit 信息是否明确
- 代码风格是否符合规范
- 是否存在语法错误
- 是否包含未定义的变量、函数的调用
- 相关依赖是否有变更（删除、升级、降级），如何测试评估
- 是否存在硬编码
- 是否有优化空间（抽象、简洁）KISS

关于发布前的检查
---
- [[一些规范#关于如何 review code]]
- 检查 release 的代码是否符合预期
- 对比发布 tag 和 release 代码是否有差异（有可能漏打 tag）
- 检查依赖，包括但不限于配置中心，数据库，运维编排
